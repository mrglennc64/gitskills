# Agile Product Owner Skill Summary

This resource provides a comprehensive toolkit for managing product backlogs and executing sprints using Scrum methodologies. Here are the key components:

## Core Workflows

The skill covers five primary workflows: user story generation, acceptance criteria writing, epic breakdown, sprint planning, and backlog prioritization. Each workflow includes validation steps to ensure quality deliverables.

## Key Distinguishing Features

The approach emphasizes "capacity math that aligns with reality" by calculating sprint capacity through velocity multiplied by an availability factor, rather than optimistic estimates. It also scales acceptance criteria requirements to story size—larger stories require more criteria to avoid under-specification.

## User Story Standards

Stories follow the template: "As a [persona], I want to [action], so that [benefit]." The tool provides templates for features, improvements, bug fixes, and enablers, each suited to different scenario types.

## Acceptance Criteria Format

Stories use Given-When-Then structure: "Given [precondition], When [action], Then [outcome]." Minimum acceptance criteria counts range from 3-4 for 1-2 point stories to requiring story splitting at 13+ points.

## Epic Splitting Techniques

Five concrete patterns help break down epics: by workflow step, persona, data type, operation (CRUD), and happy path prioritization.

## Planning & Metrics

Sprint capacity = velocity × availability factor, with committed work targeted at 80-85% capacity. The skill tracks velocity, commitment reliability (target >85%), and includes a Definition of Done checklist.
